Resumen

En esta práctica de laboratorio, se configuró el IDE de Arduino para programar el NodeMCU, instalando las herramientas necesarias según las instrucciones proporcionadas. Se revisó el pinout y los GPIO disponibles para su uso en la tarjeta seleccionada. Posteriormente, se desarrollaron diversos programas para controlar el encendido y apagado de un LED, comenzando con instrucciones básicas de encendido y apagado mediante código. Luego, se implementó un programa que alterna el estado del LED con retardos y se avanzó en la integración de botones para su control. Finalmente, se desarrollaron sistemas más complejos utilizando dos botones y condiciones lógicas.

Introducción

El ESP32 es una serie de microcontroladores de bajo costo y bajo consumo de energía desarrollados por Espressif, que integran capacidades de conectividad inalámbrica Wi-Fi y Bluetooth, junto con un procesador de doble núcleo. Su versatilidad y características avanzadas lo convierten en una opción popular para proyectos de Internet de las Cosas (IoT) y automatización del hogar. Considerado el sucesor del ESP8266, el ESP32 ofrece mejoras significativas, como mayor velocidad de procesamiento, más pines GPIO, soporte para Bluetooth clásico y BLE, así como una mayor eficiencia energética. Entre sus especificaciones técnicas destacan un procesador Tensilica Xtensa LX6 de 32 bits de doble núcleo que opera a 160 o 240 MHz, memoria SRAM de 520 KB, conectividad Wi-Fi con velocidades de hasta 150 Mbps y compatibilidad con diversos periféricos como ADCs, DACs, I2C, UART, SPI, y PWM. Además, su compatibilidad con el entorno de desarrollo de Arduino y MicroPython facilita su programación para usuarios de diferentes niveles de experiencia. En esta práctica de laboratorio se llevarán a cabo varios ejercicios con el objetivo de familiarizarse con la configuración y programación del ESP32. Se comenzará configurando el IDE de Arduino para programar el NodeMCU, instalando las herramientas necesarias. Posteriormente, se revisará el pinout y los GPIO disponibles en el NodeMCU seleccionado. Se desarrollarán diversos programas para controlar el encendido y apagado de un LED. Estos ejercicios permitirán comprender el funcionamiento básico de los pines GPIO, el manejo de entradas y salidas digitales, así como la lógica de programación aplicada al ESP32.

Desarrollo

Ejercicio 1: Desarrollar un programa de encendido de un LED mediante programación.

Este ejercicio consiste en encender un LED conectado al pin GPIO 23 del ESP32. Se configura el pin como salida digital usando pinMode(23, OUTPUT) en la función setup(), que se ejecuta una sola vez al iniciar el programa. Luego, en la función loop(), que se repite de forma continua, se utiliza digitalWrite(23, HIGH) para enviar una señal de 3.3V al pin, lo que mantiene el LED encendido de forma constante.

Ejercicio 2: Desarrollar un programa de apagado de un LED mediante programación.

Este ejercicio consiste en apagar un LED conectado al pin GPIO 23 del ESP32. En la función setup(), se configura el pin como salida digital mediante pinMode(23, OUTPUT). Luego, en la función loop(), que se ejecuta de forma continua, se utiliza digitalWrite(23, LOW) para enviar un nivel lógico bajo (0V) al pin, lo que mantiene el LED apagado de manera constante.

Ejercicio 3: Desarrollar un programa de encendido y apagado de un LED mediante retardos.

El ejercicio de parpadeo de un LED consiste en encender y apagar un LED de forma intermitente, creando un efecto de parpadeo. Esto se logra alternando el estado del pin GPIO 23 entre alto (HIGH) y bajo (LOW), con retardos temporales definidos entre cada cambio de estado. En este caso, el LED se mantiene encendido durante 300 milisegundos y apagado durante otros 300 milisegundos de manera continua.

Ejercicio 4: Desarrollar un sistema de encendido y apagado de un LED mediante el uso de botones.

Este ejercicio permite encender un LED utilizando un botón. El pin 21 está configurado como entrada para leer el estado del botón, y el pin 23 como salida para controlar el LED. Cuando se presiona el botón (entrada en alto), el LED se enciende; al soltarlo (entrada en bajo), el LED se apaga.

Ejercicio 5: Desarrollar un sistema de encendido y apagado de un LED mediante el uso de dos botones y una condicional OR.

Este ejercicio consiste en encender un LED conectado al pin GPIO 23 del ESP32 cuando al menos uno de los dos botones, conectados a los pines 21 y 22, está presionado. En la función setup(), se configuran los pines de los botones como entradas y el pin del LED como salida. En la función loop(), se leen los estados de los botones y, mediante un condicional OR, si al menos uno de los botones está presionado, se enciende el LED; de lo contrario, el LED permanece apagado.

Ejercicio 6: Desarrollar un sistema de encendido y apagado de un LED mediante el uso de dos botones y una condicional AND.

Este ejercicio enciende un LED conectado al pin GPIO 23 del ESP32 solo cuando ambos botones, conectados a los pines 21 y 22, están presionados al mismo tiempo. En la función setup(), se configuran los pines de los botones como entradas y el pin del LED como salida. En la función loop(), se leen los estados de los botones y, mediante un condicional AND, si ambos botones están presionados, se enciende el LED; de lo contrario, el LED se apaga.

Diseño (En la práctica solo se diseñaron los ejercicios 5 y 6)

El objetivo del ejercicio 5 es implementar un sistema en el que un LED se encienda cuando al menos uno de dos botones sea presionado. Para ello, el LED se conecta al pin GPIO 23 del ESP32 como salida digital, mientras que los botones se conectan a los pines GPIO 21 y 22 como entradas digitales. El sistema utiliza una operación lógica OR para determinar si el LED debe encenderse o apagarse. Cuando se presiona al menos uno de los botones, el LED se enciende; si ambos están liberados, el LED se apaga.

El código comienza configurando los pines en la función setup(), donde los botones se definen como entradas con pinMode(21, INPUT) y pinMode(22, INPUT), mientras que el LED se configura como salida con pinMode(23, OUTPUT). En la función loop(), se leen los estados de los botones con digitalRead(), y la condición if (estadoBoton1 || estadoBoton2) evalúa si al menos uno está activado. Si la condición se cumple, el LED se enciende con digitalWrite(23, HIGH), y si no, se apaga con digitalWrite(23, LOW). Este proceso se ejecuta continuamente, permitiendo la respuesta inmediata a la interacción con los botones.

Por otra parte el ejercicio 6 busca implementar un sistema donde un LED solo se encienda si dos botones son presionados simultáneamente. El LED está conectado al pin GPIO 23 del ESP32, mientras que los botones están en los pines GPIO 21 y 22 como entradas digitales. Se utiliza una operación lógica AND para verificar si ambos botones están activados al mismo tiempo. Si la condición se cumple, el LED se enciende; de lo contrario, permanece apagado.

En el código, la función setup() configura los pines, definiendo los botones como entradas con pinMode(21, INPUT) y pinMode(22, INPUT), y el LED como salida con pinMode(23, OUTPUT). En loop(), se leen los estados de los botones con digitalRead(), y la condición if (estadoBoton1 && estadoBoton2) se encarga de comprobar si ambos están presionados. Si es así, el LED se activa con digitalWrite(23, HIGH), y si no, se apaga con digitalWrite(23, LOW). Esta estructura permite que el sistema responda de manera precisa a la interacción de los botones y demuestre la aplicación de la operación lógica AND.

Construcción del Circuito

Ejercicio 1: Encendido de un LED


// Configuración inicial del ESP32
void setup() {
    pinMode(23, OUTPUT); // Configura el pin GPIO 23 como salida digital
}

void loop() {
    digitalWrite(23, HIGH); // Enciende el LED conectado al pin 23 enviando un nivel alto (3.3V)
}

Ejercicio 2: Apagado de un LED


// Configuración inicial
void setup() {
    pinMode(23, OUTPUT);  // Configura el pin GPIO 23 como salida digital
}

void loop() {
    digitalWrite(23, LOW);  // Envía un nivel lógico bajo (0V) al pin 23, apagando el LED
}

Ejercicio 3: Parpadeo de un LED


// Configura el pin 23 como salida
void setup() {
    pinMode(23, OUTPUT);
}

void loop() {
    digitalWrite(23, HIGH); // Enciende el LED
    delay(300);             // Espera 300 ms

    digitalWrite(23, LOW);  // Apaga el LED
    delay(300);             // Espera 300 ms
}

Ejercicio 4: Encendido de un LED con botón


void setup() {
    pinMode(23, OUTPUT); // Configura el pin 23 como salida (LED)
    pinMode(21, INPUT);  // Configura el pin 21 como entrada (botón)
}

void loop() {
    digitalWrite(23, digitalRead(21)); // Enciende/apaga el LED según el botón
}

Ejercicio 5: Condicional OR


void setup() {
    pinMode(23, OUTPUT); // LED como salida
    pinMode(21, INPUT);  // Botón 1 como entrada
    pinMode(22, INPUT);  // Botón 2 como entrada
}

void loop() {
    int estadoBoton1 = digitalRead(21); // Lee el estado del botón 1
    int estadoBoton2 = digitalRead(22); // Lee el estado del botón 2

    if (estadoBoton1 || estadoBoton2) {  // Si algún botón está presionado
        digitalWrite(23, HIGH); // Enciende el LED
    } else {
        digitalWrite(23, LOW); // Apaga el LED
    }
}

Ejercicio 6: Condicional AND


void setup() {
    pinMode(23, OUTPUT); // LED como salida
    pinMode(21, INPUT);  // Botón 1 como entrada
    pinMode(22, INPUT);  // Botón 2 como entrada
}

void loop() {
    int estadoBoton1 = digitalRead(21); // Lee el estado del botón 1
    int estadoBoton2 = digitalRead(22); // Lee el estado del botón 2

    if (estadoBoton1 && estadoBoton2) {  // Si ambos botones están presionados
        digitalWrite(23, HIGH); // Enciende el LED
    } else {
        digitalWrite(23, LOW); // Apaga el LED
    }
}

Resultados

En los primeros ejercicios, se verificó que el LED podía encenderse y apagarse de manera constante al asignar niveles lógicos alto y bajo a su pin de conexión. También se logró que el LED parpadeara utilizando retardos, lo que demostró el uso adecuado de la función delay() para generar cambios de estado en intervalos regulares. En los ejercicios con botones, el LED respondió correctamente a las interacciones del usuario. Se observó que podía encenderse y apagarse al presionar un solo botón, así como mediante combinaciones lógicas con dos botones, utilizando operadores OR y AND. Estos resultados evidencian que se logró utlizar el ESP32 para el manejo de diferentes señales de entrada, así como su configuración inicial

Conclusiones

En conclusión, este ejercicio nos permitió aprender los pasos iniciales para programar el ESP32, familiarizándonos con la configuración de pines como entrada y salida, así como con el uso de condicionales y el leguaje de programación. Además, nos ayudó a comprender cómo trabajar con el IDE de Arduino, una herramienta fundamental para desarrollar proyectos más complejos en el futuro.
