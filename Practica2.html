Práctica 2: “Wifi ESP32 – Sensores Digitales, Analógicos e Inteligentes”

Integrantes: Bernardo Bojalil Lorenzini, Fernando Agustín Hernández Rivas

Fecha: 14/02/2025

Primer parcial

Resumen

El objetivo de esta práctica es conocer, identificar y verificar el funcionamiento de sensores digitales, analógicos e inteligentes mediante su integración con el ESP32. Se trabajará con un botón (sensor digital), un potenciómetro (sensor analógico), un DHT22 (sensor inteligente) y un sensor PIR para la detección de movimiento. Además, se desarrollará un programa para monitorear el sensor PIR. Esta práctica permitirá comprender el funcionamiento y las aplicaciones de distintos tipos de sensores en sistemas embebidos.

Introducción

Los sensores son componentes fundamentales en los sistemas embebidos y la automatización, ya que permiten la adquisición de datos del entorno en tiempo real para su posterior procesamiento y toma de decisiones. En esta práctica, se trabajará con sensores digitales, analógicos e inteligentes, cada uno con características específicas que determinan su aplicación. Los sensores digitales miden una magnitud física, como la temperatura, la presión, la luz o la humedad, y cuantifican esa medición en formato digital, operando en dos estados lógicos (alto o bajo). Los sensores analógicos miden una cantidad física y proporcionan una señal de salida proporcional a lo medido. Los sensores inteligentes recogen datos del entorno y utilizan recursos informáticos integrados para procesarlos y realizar funciones predefinidas antes de transmitirlos. Estos se aplican en redes inteligentes, reconocimiento en campo de batalla, exploración y diversas aplicaciones científicas. El sensor PIR mide la luz infrarroja radiada por los objetos en su campo de visión, ajustando el tiempo de activación de salida (3s a 300s) y la sensibilidad (3m a 7m). Este sensor es clave en la detección de movimiento mediante cambios en la radiación infrarroja y se usa ampliamente en seguridad y automatización.

Desarrollo

Ejercicio 1: Desarrollar un programa de encendido de un LED con un botón (sensor digital).

Este ejercicio consiste en utilizar un botón y un LED con un microcontrolador. El botón está conectado al puerto GPIO 33 y el LED al puerto GPIO 32. En el código, se lee el estado del botón; si está presionado (estado 1), el LED se enciende, y si no lo está (estado 0), el LED se apaga.

Ejercicio 2: Desarrollar un programa de encendido de un LED con un potenciómetro (sensor analógico).

Este ejercicio usa un potenciómetro para controlar un LED. El potenciómetro está conectado al puerto GPIO 33 y el LED al puerto GPIO 32. El valor analógico del potenciómetro se lee continuamente. Si el valor es mayor o igual a 2048 (mitad del rango de 0-4095), el LED se enciende; de lo contrario, se apaga.

Ejercicio 3: Control de LED basado en la Humedad con Sensor DHT22 (sensor inteligente).

Este ejercicio utiliza un sensor DHT22 para medir la temperatura y la humedad. El sensor está conectado al puerto GPIO 33 y el LED al puerto GPIO 32. El código lee cada segundo la temperatura y humedad, imprimiendo los valores en el monitor serial. Si la humedad es superior al 50%, el LED se enciende, y si es menor o igual, se apaga.

Ejercicio 4: Detección de Movimiento con Sensor PIR (sensor digital).

Este ejercicio utiliza un sensor PIR para detectar movimiento. El sensor está conectado al puerto GPIO 33 y el LED al puerto GPIO 32. El código detecta el movimiento y enciende el LED, mientras imprime un mensaje en el monitor serial cuando detecta o deja de detectar movimiento. Si no hay movimiento, el LED permanece apagado.

Diseño (En la práctica solo se diseñó el ejercicio 4)

El objetivo de este ejercicio es implementar un sistema que detecte movimiento mediante un sensor PIR y encienda un LED como indicativo de detección. El sensor PIR se conecta a un puerto GPIO de la placa, y el LED se utiliza para proporcionar una señal visual de que el movimiento ha sido detectado. Además, el sistema enviará un mensaje al monitor serial cuando se detecte o termine el movimiento. En el diagrama de conexión, el sensor PIR tiene tres pines: VCC, GND, y OUT. El pin VCC se conecta al pin de alimentación (5V) de la placa y el pin GND a tierra. El pin OUT, que es la salida del sensor, se conecta al pin GPIO 33 de la placa, que se utilizará como entrada digital para leer el estado del sensor. El LED se conecta al pin GPIO 32 de la placa como salida digital. El ánodo del LED se conecta al pin GPIO 32 y el cátodo a tierra, con un resistor en serie para limitar la corriente al LED, si es necesario.

El código (ubicado en la sección Construcción del Circuito) comienza definiendo dos pines: ledPin (pin GPIO 32) para controlar el LED y inputPin (pin GPIO 33) para leer el estado del sensor PIR. Además, se define una variable pirState que se utilizará para almacenar el estado del sensor (si está detectando movimiento o no) y la variable val que leerá el valor del sensor. En la función setup(), los pines se configuran adecuadamente. El pin del LED se configura como salida usando pinMode(ledPin, OUTPUT) y el pin del sensor PIR como entrada con pinMode(inputPin, INPUT). Además, se inicia la comunicación serial con Serial.begin(115200), lo que permitirá enviar mensajes al monitor serial. Dentro de la función loop(), el programa constantemente lee el estado del sensor PIR con la función digitalRead(inputPin). Si el valor leído es HIGH, significa que el sensor ha detectado movimiento, por lo que el LED se enciende mediante digitalWrite(ledPin, HIGH) y se imprime el mensaje "¡Movimiento detectado!" en el monitor serial. Si el estado del sensor es LOW (sin movimiento), el LED se apaga utilizando digitalWrite(ledPin, LOW) y se muestra el mensaje "¡Movimiento terminado!". La variable pirState se utiliza para evitar que los mensajes de detección y terminación de movimiento se repitan constantemente. Si el movimiento acaba de ser detectado (cuando pirState es LOW), se actualiza el estado de pirState a HIGH y viceversa cuando el movimiento cesa. Esto asegura que los mensajes en el monitor serial solo se envíen una vez cuando el estado del movimiento cambia. Este ciclo se repite continuamente debido a la función loop(), que permite que el sistema esté siempre atento a los cambios en el estado del sensor PIR.

Construcción del Circuito

Ejercicio 1: Sensor digital (botón)


//Puertos GPIO
int boton=33;
int led=32;

//Variables status
int status_boton=0;

void setup() {
  // put your setup code here, to run once:
  pinMode(led, OUTPUT); // Configurar puerto 32 como salida
  pinMode(boton, INPUT); // Configurar puerto 33 como entrada
}

void loop() {
  // put your main code here, to run repeatedly:
  status_boton = digitalRead(boton);

  if(status_boton == 1){
    digitalWrite(led, HIGH);
  }
  else{
    digitalWrite(led, LOW);
  }
}

Ejercicio 4: Sensor digital (PIR)


int ledPin = 32;                // LED en el pin 32
int inputPin = 33;              // Sensor PIR en el pin 33
int pirState = LOW;             // Inicialmente, no hay movimiento detectado
int val = 0;                    // Variable para leer el estado del pin

void setup() {
  pinMode(ledPin, OUTPUT);      // Declarar LED como salida
  pinMode(inputPin, INPUT);     // Declarar sensor como entrada

  Serial.begin(115200);
}

void loop() {
  val = digitalRead(inputPin);  // Leer el valor de entrada
  if (val == HIGH) {            // Comprobar si la entrada es ALTA
    digitalWrite(ledPin, HIGH); // Encender el LED
    if (pirState == LOW) {
      // Se acaba de detectar movimiento
      Serial.println("¡Movimiento detectado!");
      pirState = HIGH;
    }
  } else {
    digitalWrite(ledPin, LOW); // Apagar el LED
    if (pirState == HIGH) {
      // Se acaba de detener el movimiento
      Serial.println("¡Movimiento terminado!");
      pirState = LOW;
    }
  }
}

Resultados

Todos los ejercicios se realizaron con éxito y funcionaron de acuerdo a lo planeado. En cada uno de ellos, los sensores respondieron correctamente a las entradas, y los LEDs se encendieron o apagaron según las condiciones establecidas en el código. En el ejercicio de detección de movimiento con sensor PIR (el único diseñado por el equipo), se presentó un inconveniente con el sensor. El LED conectado al sistema parpadeaba de manera constante, lo que no era el comportamiento esperado. Tras revisar las conexiones y el código, se determinó que el sensor PIR estaba dañado. Este fallo provocaba lecturas erroneas, lo que resultaba en el parpadeo del LED sin una detección real de movimiento. Una vez identificado el problema, el sensor fue reemplazado por uno funcional, y el sistema volvió a operar correctamente, con el LED encendiéndose únicamente cuando el movimiento era detectado y apagándose al cesar el mismo.

Conclusiones

En esta práctica, exploramos el funcionamiento de sensores digitales, analógicos e inteligentes, comprendiendo su relevancia para la adquisición y procesamiento de datos del mundo físico. Estos dispositivos son fundamentales en nuestra vida cotidiana, presentes en aplicaciones como sistemas de seguridad, electrodomésticos inteligentes, monitoreo ambiental, domótica y dispositivos médicos, entre otros.

A través de la implementación y simulación de los ejercicios, comprendimos la diferencia entre las lecturas digitales, que entregan valores binarios (encendido/apagado, alto/bajo), y las analógicas, que ofrecen un rango continuo de valores, permitiéndonos interpretar fenómenos como la temperatura, la humedad o la luz. Esta distinción es esencial para convertir variables del mundo real en datos que los sistemas computacionales pueden analizar y procesar.

Además, esta práctica resalta la importancia de los sensores como puentes entre el mundo físico y el digital, facilitando la creación de soluciones a problemas reales mediante sistemas automatizados. Comprender cómo obtener, interpretar y actuar según estas lecturas es clave para desarrollar proyectos que impacten positivamente a la sociedad.
