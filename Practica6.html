Práctica 6: “Configuración de servidor para Red Local”

Integrantes: Bernardo Bojalil Lorenzini, Fernando Agustín Hernández Rivas

Fecha: 26/04/2025

Tercer parcial

Resumen

En esta práctica se busca conocer, investigar e identificar los pasos necesarios para configurar una red local y permitir el acceso a recursos de un equipo en red. Se realiza la instalación de XAMPP para montar un servidor web local, habilitando los servicios de Apache y configurando el acceso a través del puerto 80. También se identifica la dirección IP asignada al equipo servidor dentro de la red y se comprueba el acceso al sitio web montado desde tres dispositivos distintos conectados a la misma red, asegurando así la correcta configuración del entorno de red y servidor.

Introducción

En el ámbito del desarrollo web y la administración de sistemas, la capacidad de configurar entornos locales que simulen condiciones reales de producción es esencial para el diseño, prueba y despliegue eficiente de aplicaciones. Una red local (LAN, por sus siglas en inglés) permite la interconexión de múltiples dispositivos dentro de un área geográfica limitada, facilitando el intercambio de recursos y la comunicación entre equipos. Esta infraestructura es fundamental no solo en entornos empresariales, sino también en contextos educativos y domésticos, donde se requiere compartir archivos, impresoras y servicios de manera eficiente.​ Para implementar y gestionar servicios web en una red local, es común utilizar herramientas que simplifiquen la instalación y configuración de los componentes necesarios. XAMPP es una de las soluciones más populares en este sentido. Se trata de un paquete de software libre que incluye el servidor web Apache, la base de datos MariaDB, y los intérpretes para lenguajes de programación como PHP y Perl. Su diseño multiplataforma y su facilidad de instalación lo convierten en una opción ideal para desarrolladores que buscan establecer un entorno de pruebas local sin complicaciones. ​ La configuración de un servidor local con XAMPP permite a los usuarios montar sitios web y aplicaciones en su propio equipo, facilitando el desarrollo y la prueba de funcionalidades antes de su implementación en un entorno de producción. Además, al habilitar el acceso a través del puerto 80 y configurar adecuadamente el firewall, es posible permitir que otros dispositivos dentro de la misma red local accedan a los recursos alojados en el servidor, emulando así un entorno colaborativo o de cliente-servidor.

Ejercicio: Servidor Apache

Para iniciar la práctica, se abrió la aplicación XAMPP y en la pestaña "Manage Servers" se activó el servicio Apache Web Server, encargado de ejecutar el servidor local. Una vez en estado "Running", se accedió a la carpeta htdocs dentro del directorio de instalación de XAMPP, donde se creó un archivo llamado index.html con una estructura básica en HTML que mostraba el mensaje "El servidor local funciona.". Posteriormente, se verificó el funcionamiento del servidor local abriendo un navegador web y accediendo a http://localhost, donde se visualizó correctamente el mensaje programado. Para permitir el acceso desde otros dispositivos, se identificó la dirección IP local del equipo ejecutando el comando ipconfig getifaddr en0 en la Terminal, obteniendo la IP 172.18.120.247. Finalmente, se comprobó el acceso al servidor ingresando la dirección IP desde tres dispositivos diferentes conectados a la misma red: dos celulares y una laptop. En todos los casos, el sitio web se mostró correctamente, confirmando que el servidor Apache y la configuración de red local funcionaban adecuadamente.

Simulación

Ejercicio

Diseño

El diseño de la práctica se basó en la creación de una estructura básica de un sitio web utilizando el lenguaje de marcado HTML. El objetivo principal fue montar un archivo sencillo que permitiera verificar de manera rápida y clara el funcionamiento del servidor local configurado con XAMPP.

Para ello, se elaboró un archivo llamado index.html dentro de la carpeta htdocs, la cual actúa como el directorio raíz del servidor Apache. El archivo HTML contiene una estructura mínima pero funcional, iniciando con la declaración <!DOCTYPE html>, seguida de las etiquetas principales <html>, <head>, <title>, <body> y <h1>. En el encabezado (<head>), se incluyó el título de la página, nombrado "Práctica 6 - Bernardo & Fernando", mientras que en el cuerpo (<body>) se colocó un encabezado de primer nivel (<h1>) que muestra el mensaje "El servidor local funciona."

Este diseño sencillo permite confirmar de manera inmediata que el servidor web está activo y que los dispositivos conectados en la misma red pueden acceder correctamente al sitio alojado.

Resultados

Al finalizar la configuración del servidor local utilizando XAMPP, se comprobó que el servicio Apache funcionaba correctamente. El sitio web creado en el archivo index.html fue accesible de manera local escribiendo http://localhost en el navegador del equipo donde se instaló XAMPP. Posteriormente, para validar el acceso desde otros dispositivos dentro de la misma red, se identificó la dirección IP del equipo servidor. Ingresando esta IP en el navegador web de distintos dispositivos, se pudo acceder correctamente al sitio web alojado en el servidor local. Esta prueba confirmó que la configuración fue exitosa, permitiendo que el servidor montado en el equipo local fuera visible desde teléfonos, laptops y los dispositivos conectados a la misma red.

Conclusiones

La práctica permitió comprender de manera práctica cómo configurar y utilizar un servidor local mediante el uso de XAMPP. A través de la instalación y activación del servicio Apache, se logró montar un sitio web básico que fue accesible tanto desde el equipo servidor como desde otros dispositivos conectados a la misma red, demostrando el correcto funcionamiento del entorno de red local. El uso de XAMPP simplificó considerablemente el proceso de creación de un servidor, ya que integra en una sola herramienta los servicios fundamentales necesarios para el desarrollo web, como Apache, MariaDB y PHP. Esta facilidad permite a los usuarios crear, probar y modificar sitios web de forma segura en un entorno controlado, sin necesidad de utilizar un servidor real conectado a Internet. Comparado con un servidor real, un servidor local funciona de manera similar en cuanto a servir archivos y gestionar peticiones, pero se limita a un ámbito privado, generalmente dentro de una misma red local. Mientras que los servidores reales están diseñados para ser accesibles públicamente y deben enfrentar desafíos como la seguridad externa, la estabilidad continua y el tráfico de múltiples usuarios simultáneos, los servidores locales son principalmente herramientas de desarrollo y prueba, ofreciendo un entorno seguro donde los errores pueden corregirse antes de la publicación final.
