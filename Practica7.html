Práctica 7: “Redes de computadora con Switches y Cables”

Integrantes: Bernardo Bojalil Lorenzini, Fernando Agustín Hernández Rivas

Fecha: 28/04/2025

Tercer parcial

Resumen

La práctica tiene como propósito conocer, identificar y comprobar el funcionamiento de una red de área local utilizando switches y routers. Se deben elaborar dos redes: una empleando un switch y otra un router, conectando al menos tres computadoras y un servidor, utilizando asignación automática de IP y seleccionando el tipo de cable adecuado. Posteriormente, se realizan pruebas de conectividad mediante comandos como PING y se verifica el acceso a la página web del servidor. Finalmente, ambas redes deben unirse y se debe comprobar el intercambio de archivos entre los dispositivos.

Introducción

Una red de área local (LAN, por sus siglas en inglés) es un sistema de interconexión de computadoras y otros dispositivos dentro de un espacio geográfico reducido, como una oficina, un laboratorio, un edificio o incluso un hogar. Estas redes permiten compartir datos, aplicaciones y recursos físicos, como impresoras, servidores y acceso a Internet, de manera rápida, segura y eficiente. Gracias a su alta velocidad de transmisión y su bajo coste de implementación y mantenimiento, las LANs se han convertido en una infraestructura indispensable en entornos empresariales, académicos y domésticos. El diseño y correcto funcionamiento de una LAN requiere de dispositivos esenciales, principalmente los switches y los routers. Un switch es un equipo de red que opera principalmente en la capa 2 del modelo OSI (capa de enlace de datos) y que se encarga de interconectar dispositivos dentro de una misma red local. Utiliza direcciones MAC (Media Access Control) para dirigir los datos de manera precisa a su destino dentro de la red, minimizando las colisiones y mejorando el rendimiento general. A través del uso de switches, es posible lograr comunicaciones simultáneas entre varios dispositivos, optimizando así el tráfico de red y la eficiencia de los recursos. En contraste, un router es un dispositivo que opera principalmente en la capa 3 del modelo OSI (capa de red) y tiene como función principal interconectar diferentes redes, como una LAN con una red de área amplia (WAN), por ejemplo, Internet. Además de enrutar el tráfico de datos entre redes distintas, los routers suelen incorporar funciones adicionales como servidores DHCP (Dynamic Host Configuration Protocol), firewalls y puntos de acceso inalámbricos. El DHCP permite asignar automáticamente direcciones IP a cada dispositivo de la red, lo cual simplifica enormemente la administración y configuración de la infraestructura, evitando conflictos de direcciones y facilitando la expansión de la red conforme sea necesario. La correcta verificación de la conectividad en una LAN es fundamental para asegurar su funcionamiento adecuado. Para ello, se emplean herramientas de diagnóstico como el comando PING. Este comando envía paquetes de solicitud de eco (ICMP) a un dispositivo objetivo y mide el tiempo que tarda en recibir una respuesta, permitiendo determinar si existe conectividad, identificar posibles problemas de latencia o pérdidas de paquetes, y evaluar la estabilidad de la red.

Desarrollo

Ejercicio 1: Switch

Para la elaboración de la red, en primer lugar se seleccionaron los equipos necesarios: un switch, cuatro computadoras (tres como clientes y una destinada como servidor) y el cableado apropiado. Se utilizó cable de red tipo cable par trenzado categoría 5e (Cat 5e), que es adecuado para conexiones de hasta 1 Gbps dentro de redes LAN. Para este tipo de conexión, dado que los dispositivos (computadoras y switch) son diferentes, se emplearon cables de red directos (cableado recto, T568A en ambos extremos). A continuación, se procedió a conectar cada computadora al switch utilizando los cables directos, asegurándose de que cada dispositivo estuviera correctamente energizado y conectado en un puerto disponible del switch. El servidor también fue conectado de la misma manera al switch. Para la configuración de la red, se habilitó la opción de asignación automática de direcciones IP mediante DHCP. En este caso, se aprovechó el servidor como origen de la asignación DHCP, o en su defecto, se conectó temporalmente un router con servicio DHCP activo para que proporcionara las direcciones IP a todos los dispositivos conectados. Una vez realizada la conexión física y la configuración básica, se verificó que cada computadora recibiera correctamente una dirección IP automática dentro del mismo rango de red. Para comprobar la conectividad entre los dispositivos, se utilizó el comando PING desde cada computadora hacia las demás, observando que los paquetes enviados fueran recibidos correctamente y que no existieran pérdidas, lo que indicó una comunicación exitosa entre todos los equipos.

Ejercicio 2: Router

Para la elaboración de la red utilizando un router, se conectaron tres computadoras cliente y una computadora servidor utilizando cable par trenzado categoría 5e (Cat 5e), empleando cables directos (T568A) para las conexiones entre los dispositivos y el router. Cada computadora fue conectada a un puerto LAN del router, asegurando una correcta conexión física. El router se configuró para asignar automáticamente direcciones IP a todos los dispositivos mediante su servidor DHCP interno, lo que permitió que cada equipo obtuviera una dirección IP dentro del mismo segmento de red sin intervención manual. Una vez verificada la obtención de IP en cada dispositivo, se realizaron pruebas de conectividad utilizando el comando PING, enviando paquetes entre las computadoras y hacia el servidor, confirmando que no existían pérdidas de datos y que la comunicación era exitosa. Finalmente, se configuró un servicio web en el servidor y, mediante un navegador web en cada computadora cliente, se accedió a la página alojada en el servidor ingresando su dirección IP, comprobando así el correcto funcionamiento de la red tanto a nivel de conexión como de servicios.

Ejercicio 3: Router + Switch

Para integrar ambas redes, se conectó un puerto del switch a un puerto LAN del router utilizando un cable directo, lo que permitió la comunicación entre los dispositivos de ambos segmentos. Además, se añadieron tres celulares a la red, conectándolos de manera inalámbrica al router y configurándolos para obtener direcciones IP automáticamente mediante DHCP. Se verificó la conectividad utilizando el comando PING entre las computadoras, el servidor y los celulares, confirmando que todos los dispositivos podían comunicarse correctamente. Finalmente, se comprobó que los celulares pudieran acceder al servidor web ingresando su dirección IP en los navegadores, asegurando que la integración de la red funcionara correctamente.

Diseño

En el primer ejercicio se conectaron cuatro laptops utilizando cables directos a un switch, siendo una de ellas configurada como servidor con el servicio de XAMPP activo. Una vez establecidas las conexiones físicas, se realizaron pruebas de conectividad utilizando el comando PING entre los dispositivos para comprobar que podían comunicarse correctamente. Posteriormente, se activó el servidor XAMPP y desde cada una de las laptops clientes se intentó acceder a la página web alojada en el servidor ingresando su dirección IP en los navegadores, verificando así la correcta comunicación y funcionamiento de la red. En el segundo ejercicio se repitió el mismo procedimiento, pero en lugar de utilizar un switch se empleó un router. Las cuatro laptops, incluida la que funcionó como servidor con XAMPP, se conectaron directamente al router mediante cables directos. El router asignó automáticamente direcciones IP a cada dispositivo a través de su servicio DHCP. Se realizaron las mismas pruebas de conectividad con el comando PING y se comprobó el acceso al servidor web desde todas las laptops, confirmando el correcto funcionamiento de la red a través del router. Finalmente, en el tercer ejercicio se integraron ambas redes conectando un puerto del switch a un puerto LAN del router. Además de las cuatro laptops, se añadieron tres celulares que se conectaron de manera inalámbrica al router. Se realizaron pruebas de conectividad utilizando PING entre todos los dispositivos, asegurando que tanto las laptops como los celulares podían comunicarse entre sí. También se verificó el acceso a la página web del servidor desde todos los dispositivos, confirmando así la correcta integración y operación de la red completa.

Resultados

En todos los ejercicios realizados, los resultados fueron exitosos. Durante las pruebas de conectividad utilizando el comando PING, todos los dispositivos —tanto laptops como celulares— respondieron correctamente, sin pérdidas de paquetes ni tiempos de espera excedidos, lo que demostró una comunicación estable dentro de la red en cada configuración. Al activar el servidor XAMPP en la laptop designada como servidor, todas las laptops clientes lograron acceder sin inconvenientes a la página web alojada, ingresando la dirección IP correspondiente en sus navegadores. En el caso de la integración de redes, después de conectar el switch al router y añadir los celulares vía Wi-Fi, también se comprobó que estos dispositivos podían comunicarse correctamente mediante PING y acceder al servidor web sin dificultades. Los resultados confirman que las conexiones físicas, la asignación automática de direcciones IP y la configuración de los servicios de red se realizaron de manera adecuada.

Conclusiones

La práctica realizada permitió comprobar la importancia y utilidad de las redes locales (LAN) en la comunicación eficiente entre múltiples dispositivos dentro de un mismo entorno. Se observó que elementos como el switch y el router son fundamentales para organizar y gestionar el tráfico de datos: el switch facilita la conexión directa entre varios dispositivos dentro de la misma red, mientras que el router permite, además, gestionar el direccionamiento de IPs y conectar redes diferentes, integrando tanto conexiones alámbricas como inalámbricas. Asimismo, quedó demostrado que la correcta selección y utilización de los cables de red es esencial para garantizar una comunicación estable y de alta velocidad. Gracias a estas tecnologías, es posible compartir servicios, recursos y acceder a información de manera inmediata dentro de una organización o espacio de trabajo, haciendo que las redes locales sean indispensables para el funcionamiento eficiente de sistemas informáticos modernos. Además, las pruebas realizadas evidenciaron la facilidad con la que pueden expandirse las redes para integrar más dispositivos, manteniendo la comunicación efectiva y la disponibilidad de los servicios.
